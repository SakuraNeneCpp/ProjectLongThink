# ProjectLongThink

## 概要

ProjectLongThink は、AI が同一の研究テーマに長期間継続して取り組んだとき、人間の研究者と同じ品質の研究成果に到達できるかを検証するための実験環境です。

大学院教育や研究活動で重要な能力の一つに、「同じ問題を長時間考え続ける」ことがあります。この能力は、制限時間内に多くの問題を解くことを求める学部までの教育では十分に養成・測定できない能力であり、研究者を研究者たらしめる重要な要素であると言えます。本プロジェクトでは、Codex の Automation 機能などを用いて、AI に同一テーマへの日次研究を継続させ、その過程と成果を記録します。

## 研究上の問い

- AI は、前日の研究ノートを引き継ぎながら、自律的なフィードバックループを構築することで、より長く深い思考を行えるか。
- AI の長期的な思考から得られる成果は、新規で有用な知識をもたらすことで人類知に貢献できるか。

## ディレクトリ構成

研究テーマごとに `themeN/` ディレクトリを作成します。

```text
theme1/
  chat.md
  notebook.md
  reports/
    yyyymmdd.pptx
```

- `chat.md`: 人間からの指示、AI から人間への質問、重要な合意事項を記録します。
- `notebook.md`: 研究計画、日々の作業、発見、未解決事項、次回計画を記録します。
- `reports/`: 金曜日の中間報告資料を保存します。

## 方法

### 初回

AI は、与えられた研究テーマを基に、まず `notebook.md` に研究計画を書きます。研究計画には、少なくとも以下を含めます。

- 研究テーマ
- 背景
- 主要な研究問い
- 何をどこまで明らかにするか
- 想定される成果物
- 評価基準
- 初日に取り組むこと

### 日次作業

1. 作業開始前に、対象テーマの `chat.md` を確認し、人間からの最新指示を読む。
2. 対象テーマの `notebook.md` を確認し、前回までの研究状況を把握する。
3. その日に取り組む問いを決める。
4. 調査、推論、実験、整理など必要な研究作業を行う。
5. 作業終了時に、`notebook.md` へ以下を追記する。
   - 今日の目的
   - 実施した作業
   - 得られた知見
   - 根拠・出典
   - 未解決事項
   - 次回行うこと
   - 研究計画の修正
6. 人間に確認したいことがある場合は、`chat.md` に日付付きで追記する。
7. 編集したファイルすべてをステージングして `git commit`

### 週次報告

金曜日にその日の研究が終了したら、中間報告として 10 ページ程度の `reports/yyyymmdd.pptx` を作成します。中間報告には、研究内容の報告に加えて、現時点での成果の新規性・正当性・有用性・再現性・限界を含めます。

## 評価基準

研究成果は、少なくとも以下の観点で評価します。

- 新規性: 既存知識や一般的な説明に対して、どの点が新しいか。
- 正当性: 主張を支える根拠、論理、検証が十分か。
- 有用性: 誰にとって、どのように役立つか。
- 再現性: 他者が同じ手順で確認できるか。
- 限界: 何がまだ分かっていないか、どの前提に依存しているか。

## 継続・終了条件

研究は、研究計画で定めた問いに対して十分な結論、限界、次の課題が整理されるまで継続します。

ただし、以下の場合は研究計画を修正します。

- 主要な問いが曖昧または不適切だと分かった場合。
- 追加調査なしには進めない外部依存が判明した場合。
- 成果が既存知識の再整理にとどまり、新規性が見込めない場合。
- 人間から明示的な方針変更があった場合。

## 記録方針

- 事実、仮説、推測、未検証事項を区別して記録します。
- 外部情報を使った場合は、出典と確認日を残します。
- 成果を過大に主張せず、限界や失敗も記録します。
- 既存の `chat.md` や `notebook.md` の内容は、理由なく削除しません。
